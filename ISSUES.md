## Backlog

### UI/UX issues

- Starting a route (from the Map Screen or the floating widget) should open the same modal as from the route list to expose every options possible
- The "roaming" icon when expanded isn't correct. It should expand on the left, not the right. The colors of the icon should match the route one: "compass" should be green, background black, "stop" should be red, background black.
- When clicking "generate" in the roaming option, then "start" the road is re-generated before starting, so the user end up with a different route than the one he previewed on the map when clicking "start".
- When starting roaming, "pause" isn't available from the floating widget, it should be. A roaming is just an ephemeral route. it should behave exactly the same.
