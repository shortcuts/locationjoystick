Settings

Configure speed profiles, GPS realism, widget controls, roaming defaults, and data export/import. The Settings hub organizes configuration into four sub-screens: Movement & GPS, Menus, Favorites & Routes, and Roaming.

  • Movement & GPS: Edit Slow Walk, Walk, Run, Bike, and Drive speed presets used by the joystick, routes, and roaming. Toggle GPS realism features independently: bearing hold, altitude drift, warm-up envelope, satellite count, signal dropouts.
  • Menus: Choose which features appear in the floating widget and on the map, and drag to reorder them.
  • Favorites & Routes: Enable curated hot locations and routes for quick access.
  • Roaming: Configure default area, distance, speed profile, road-following, and auto-return behavior for roaming sessions.

Movement & GPS

Combines speed profile configuration with GPS realism settings. Configure the movement speeds and customize how the spoofed signal behaves.

Speed Profiles

Five presets set the movement speed for the joystick, route replay, and roaming. Edit any of them; changes apply immediately. A km/h ↔ mph switch at the top converts every field in the app at once.

ProfileDefaultPace
Slow Walk 1.1 km/h Strolling, browsing shopfronts
Walk 2.0 km/h Regular walking pace
Run 8.0 km/h Jogging — follows footpaths when road-following is on
Bike 15.0 km/h Cycling — follows bike lanes when road-following is on
Drive 54.0 km/h Driving — well above the anti-cheat threshold below

There's no upper limit — type any speed you want. Values only get floored, never capped: anything entered below 0.04 km/h snaps up to it, since a true 0 would mean the simulated device never moves at all.

Each profile also has a checkbox controlling whether it's included when the widget's Speed cycle button cycles through presets — Walk, Run, and Bike are on by default; Slow Walk and Drive are opt-in. This only affects cycling — all five stay editable here regardless.

Anti-cheat warning

Set any profile above 28.8 km/h (17.9 mph) and a warning appears inline below that field:

"Speed exceeds 8 m/s — may trigger anti-cheat in some games"

It's advisory only — nothing is blocked, capped, or sent anywhere. The threshold sits just above any realistic human pace on foot, which is what many location-aware games watch for. Each profile is checked independently; lower it below the threshold to clear the warning, or ignore it if that's not a concern for what you're using it with.

Location Randomness

Adds small random variations to each location update to simulate natural GPS drift. Set any value to 0 to disable that dimension of variation.

OptionAppliesWhat it does
Idle radius Stationary Max random offset while standing still. 1–3 m reads as a real chip drifting in place.
Idle interval Stationary Seconds between new offsets while still. Shorter = jitterier trace.
Moving radius Moving Max random offset while moving. Keep below idle radius to avoid unnatural jumps.
Moving interval Moving Seconds between new offsets while moving.
Idle speed variation Stationary Random ±% on reported speed at rest. 5–10% mimics measurement noise; 0 reads as perfectly still.
Moving speed variation Moving Random ±% on reported speed while moving, so it doesn't read as a flat constant.

GPS Realism

Independent toggles that make the spoofed GPS signal behave more like a real hardware chip. Some apps and games inspect these fields to detect mock providers — enabling all options reduces the chance of detection.

OptionDefaultWhat it does
Hold bearing when stationary On Keeps the last heading instead of snapping to north on stop — a sudden reset is a common tell.
Vary altitude On Drifts the reported altitude slightly rather than holding one constant height.
GPS warm-up simulation Off Starts each session with degraded accuracy that sharpens over ~30 s, like a cold GPS fix.
Realistic satellite count On Reports 7–14 visible satellites, 6–12 in use. Zero satellites is a spoofing tell some apps check for.
Suspended mocking Off Briefly drops updates (~2 s every ~10 s) to mimic real signal dropouts. Off automatically during route replay.

Map

Controls how the in-app map behaves while spoofing is active.

OptionDescription

Menus

Appearance

A Light mode switch lets you flip the whole app to a bright, high-contrast theme instead of the default dark orange-on-black look — handy for reading the screen outdoors in direct sunlight. The change applies immediately everywhere in the app.

Both surfaces follow the same order, so reordering once keeps everything consistent.

Roaming

Configure default area, distance, speed profile, road-following, and auto-return behavior for roaming sessions.

OptionDescription
Radius Maximum distance from the starting point that random waypoints can be placed. Larger values spread movement further from the centre.
Route distance Total distance to walk before the roaming session ends. The session stops once the cumulative distance reaches this value.
Speed profile Which speed preset (Walk / Run / Bike) is used for roaming movement. Bike also selects the cycling profile in OSRM road routing.
Follow roads Routes each leg through OSRM so movement follows real walkable paths instead of straight lines. Falls back to straight-line if OSRM is unavailable.
Return to start After the main roaming loop completes, the simulated device walks back to the original starting position.

External services

locationjoystick is offline-first. It does not require an account and sends no analytics or telemetry. However, two optional network features make outbound requests:

ServiceWhen usedWhat is sent
OSRM
router.project-osrm.org		 Road-following routes, roaming with "Follow roads" enabled, guided route creation. Only contacted when you explicitly choose road-following. A list of latitude/longitude coordinate pairs (your waypoints or random roaming targets). No device identifiers, no account data. The public OSRM demo server is operated by the OSRM project.
Nominatim
nominatim.openstreetmap.org		 Place-name search in the route creator and the favorites map picker. Only contacted when you type a search query. The search string you type (place name or address). No GPS coordinates, no identifiers. The public Nominatim instance is operated by the OpenStreetMap Foundation.

Map tiles (OpenStreetMap via MapLibre) are fetched from tile.openstreetmap.org as you pan and zoom the map. Each tile request includes the tile coordinates (zoom level, x, y) but no personal data.

All three services are used only as needed by the feature you are actively using and can be avoided entirely by not using road-following, place search, or the map (e.g. by entering coordinates directly in Favorites).

QR transfer

Share your entire configuration between devices by scanning a single QR code — no cables, no accounts, no cloud.

  • Both devices need to be on the same Wi-Fi network.
  • The sending device shows one QR code (and a short code as a backup); the receiving device scans it, or types the code, and imports right away.
  • No cloud involved — the data travels directly between the two devices over the network.

Export / Import

Back up or restore all app data — routes, favorites, speed profiles, widget config, jitter settings, and roaming defaults.

ActionDescription
Export via QR code Shows a QR code that another device on the same Wi-Fi network can scan to receive your current configuration.
Export settings Writes a JSON backup file to device storage, which you can share or keep manually.
Import from QR code Opens the camera to scan a code shown on another device (same Wi-Fi network), then prompts to Add (merge) or Replace existing data.
Import via code Type the short code shown on the sending device instead of scanning — same Wi-Fi network required.
Import from file Opens a file picker for a previously exported JSON backup. Prompts to Add or Replace.
Import from GPS Joystick Migrates routes and favorites from a GPS Joystick export file. Imported routes are saved as straight-line segments.
Import from YAMLA Migrates favorites and speed settings from a YAMLA JSON export.

Next to the export/import buttons is a Reset all data button. It asks you to confirm, then permanently deletes all favorites, routes, and settings — without needing to clear the app's data through your phone's system settings. Your permission grants are kept, so you won't need to go through setup again.
